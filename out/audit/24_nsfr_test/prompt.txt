HOW TO COUNT SPAN FROM THE IMAGE (critical):
  A merged cell's value is printed at the VISUAL CENTRE of the entire merged region —
  NOT at the leftmost column. For an odd-span merge (span=3,5,...) the value sits exactly
  above the middle column. For an even-span merge (span=2,4,...) the value sits between
  two columns. In both cases you MUST scan both left AND right from the value.

  The ONLY reliable signal is vertical borders combined with cell contents:
  - Scan LEFT from the value: keep extending while the cell to the left is BLANK (no value,
    no dash). Stop when you hit a border AND the cell beyond it has a value OR a dash.
  - Scan RIGHT from the value: same rule — keep extending while the cell to the right is
    BLANK (no value, no dash). Stop when you hit a border AND the cell beyond it has a value
    OR a dash, OR you reach the end of the row.
  - Count all columns from left stop to right stop → that is the span.

  KEY INSIGHT — blank ≠ separate cell:
  A cell that is visually EMPTY with NO printed dash is part of the merged region if there
  is no hard vertical border separating it from a cell that has a value. Do NOT treat it as
  a standalone empty cell. Absorb it into the span.

  STEP-BY-STEP for every merged value:
  1. Find the column where the value is printed.
  2. Scan LEFT: does the adjacent cell contain a value or dash? If NO (blank) → extend span
     leftward. Repeat until you find a value/dash or the row label column.
  3. Scan RIGHT: does the adjacent cell contain a value or dash? If NO (blank) → extend span
     rightward. Repeat until you find a value/dash or the end of the row.
  4. Span = total columns covered (left extension + value column + right extension).

  EXAMPLE — value centred over col3 of 5, cols 2 and 4 are blank (no dash):
  WRONG: [col1:7711, col2:''{span=1}, col3:9964{span=1}, col4:''{span=1}, col5:192]  ← 4 cells, span=1
  WRONG: [col1:7711, col2:''{span=1}, col3-4:9964{span=2}, col5:192]                ← missed left blank
  RIGHT: [col1:7711, col2-4:9964{span=3}, col5:192]   sum=1+3+1=5=ncols ✓

  The blanks on BOTH sides of 9,964 are absorbed because neither contains a dash.

BLANK vs DASH — critical distinction:
  "-" (dash) = the bank explicitly recorded nil/zero for that specific column. Real standalone
               cell with hard borders on both sides. Always emit {"value":"-","span":1,"merge_type":"none"}.
  ""  (blank, no dash) = no printed content. Check: is there a hard vertical border on BOTH
      sides? If the adjacent cell (left or right) has a value/dash with a hard border → real
      empty cell, emit {"value":"","span":1,"merge_type":"none"}.
      If the adjacent cell is ALSO blank or contains the merged value → part of the merge,
      absorb into span. Never emit a blank GCell for a column that is inside a merged region.

MERGED BLANK CELLS:
  If a merged region has no value (greyed out, shaded, or empty):
    {"value": "", "span": N, "merge_type": "blank"}

merge_type values:
    "none"      — span=1 always
    "aggregate" — value covers N columns, not attributable to any single column
    "repeat"    — same value independently applies to each covered column
    "blank"     — merged empty/shaded region

SPAN IS DETERMINED BY VERTICAL BORDERS ONLY — not by content, not by column groups:
  The ONLY signal for whether two adjacent cells are merged is whether there is a vertical
  border line between them in the PDF.
  - NO vertical border between col N and col N+1 → they are merged, one GCell covers both
  - Hard vertical border between col N and col N+1 → they are separate cells, even if both blank

  This means:
  - A blank cell with hard borders on BOTH sides is a real standalone empty cell → span=1, value=""
  - A blank cell with NO border on one side is part of the adjacent merge → absorb into span
  - A total row where every column has its own hard border → every cell is span=1, even if blank

  Before emitting any GCell, ask: "Is there a drawn vertical line between this cell and its
  neighbour?" If YES on both sides → span=1. If NO on one side → merge with neighbour.

- Do NOT fill values left-to-right when a row is sparse. Missing vertical borders = merged cell.
- Do NOT emit values=[] for any row that has visible numbers in the PDF.
